3. CLOUDFLARE ELEMZÉS
================================================================================

A Cloudflare két problémát oldana meg egyszerre:
  1. CDN/DNS → Facebook crawler átjut
  2. R2 tárolás → képtárolás (10GB ingyen)

DE: a Cloudflare NEM hosting — kiegészítő szolgáltatás.
  - Vercel továbbra is futtatja az appot (konyha)
  - Cloudflare eléáll (pincér): DNS, CDN, védelem, képtárolás

ÖSSZEHASONLÍTÁS:
  Vercel Hobby + Cloudflare = $0/hó, 10GB kép, Facebook megy
  Vercel Pro = $20/hó, 1GB kép, Firewall rule-lal FB megoldható
  Cloudflare Pages = kiválthatná a Vercelt, de Next.js támogatás nem 100%
  Railway + Cloudflare = $5/hó, nincs beépített képtárolás

DÖNTÉS:
  Egyelőre NEM váltunk Cloudflare-re.
  Profilkép → Cloudinary (ingyenes, 25GB, platformfüggetlen).
  Cloudflare → soft launch előtt, ha kell a Facebook megosztás.

4. PROFILKÉP FELTÖLTÉS — CLOUDINARY
================================================================================

MEGVALÓSÍTVA:
  - Cloudinary fiók: dx8forkxy cloud name
  - API endpoint: /api/upload (POST, FormData)
    - Bejelentkezett felhasználónak
    - Max 5MB képfájl
    - Cloudinary-re tölt fel: 400x400px, arcra fókuszál (gravity: face)
    - Automatikus tömörítés (quality: auto)
    - Felülírja a régit (user ID = public_id)
    - Menti a photoUrl-t a User modellbe

  - Profil oldal UI:
    - "Profilkép" szekció a Statisztikák és Profil szerkesztése között
    - Kör alakú előnézet (jelenlegi kép vagy kezdőbetű)
    - "Kép választása" gomb, "Max 5MB, JPG vagy PNG" felirat

  - Megjelenítés mindenhol:
    - PlayerCard: ha van photoUrl, kép a kezdőbetű helyett
    - Játékos profil oldal (/jatekosok/[id])
    - Főoldali keresés (SearchSection)
    - API-k frissítve: /api/users és /api/users/[id] visszaadják a photoUrl-t

HIBAELHÁRÍTÁS:
  - Első hiba: "Invalid Signature" — a .env fájlban a CLOUDINARY_URL
    formátuma hibás volt (secret és cloud name összekeveredett)
  - Második hiba: "Must supply api_key" — a Next.js nem olvasta be az
    .env-t futás közben, újraindítás kellett
  - Megoldás: explicit cloudinary.config() a kódban + külön env változók:
    CLOUDINARY_CLOUD_NAME, CLOUDINARY_API_KEY, CLOUDINARY_API_SECRET

VERCEL BEÁLLÍTÁS:
  A 3 Cloudinary env változót a Vercel Dashboard-on is be kell állítani
  (Settings → Environment Variables).
